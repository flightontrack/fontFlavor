Flight On Track (FONT) is Android app that tracks aircraft position, velocity and altitude based on the information received from a pilot's smartphone. The route can be tracked in real time, analyzed, shared or exported via http://flightontrack.com website.

Current Requirements:
  • Android 4.1 or higher smartphone with gps
  • Data access over mobile network
  • FONT app installed on Android phone
Preflight:
  • In order to avoid distraction make sure all preparations and checks are done before engine start.
  • Launch the FONT application on your phone.
  • To update Pilot or Aircraft info touch the upper card of the main screen or click on airplane icon located on the bottom.
  • Make sure the Pilot field is populated with your phone number id or another pilot callsign. Edit it if needed.
  • Update Aircraft Number scanning the aircraft NFC tag if available. Alternatively, edit the Aircraft Number field manually. Only this field is required to be populated. Press Save button.
  • Set Update Frequency interval ranging from 3 seconds for flying in pattern to 5 minutes for long flights.
  • Check MultiLeg box if you are flying pattern or on a flight with multiple legs and landings.
  • Check Min Speed making sure it is set to desired speed. The default set is 40 mph meaning the actual recording starts upon reaching this speed. If the setting reads 0 the recording starts right after getting a flight number.
  • Touch the red button before you start taxiing. It is important to have a connection to your mobile network at this time. The app starts communicating with cloud remote server, requesting a new Flight ID. If the communication went out successfully the button switches to yellow Cleared For Takeoff state with the new flight number received. If the application failed to connect to the cloud and get the flight number a local temporary number is allocated. The temporary flight number is replaced with a cloud one automatically when the connection becomes available.
  • Place the phone in your aircraft where the gps signal is not blocked though it may work absolutely fine in your pocket.
Checklist:
  • Pilot callsign ID
  • Aircraft number
  • Update Frequency
  • MultiLeg
  • Min Speed set as desired
In flight:
  • No action required from the pilot in flight.
  • Upon reaching 40 mph speed on takeoff the button automatically changes from yellow to green On Flight state indicating the start of the tracking. The tracking automatically stops when flight slows down below the Min Speed on landing. If MultiLeg box is checked the application automatically allocates a new flight id and starts another flight recording on the next takeoff.
  • To stop tracking press the green button.
  • When the aircraft is taxing or stopped no trackpoints are recorded.
Shareing a flight on Facebook:
  • Your flight Google Maps route can be shared on Facebook while you are in flight in real time.
  • The flight needs to be started first and a flight number needs to be received.
  • Use Share on FB icon on Action bar in Preflight phase to share.
My Pilot Logbook:
  • Your pilot logbook is stored on the cloud server.
  • Use Logbook icon to reach your logbook records.
Changing/resetting application preferences:
  • Use Settings icon on the bottom of Action bar.
  • To revert all settings to default values press Reset to Defaults button.
  • Site: shows the url where your routes can be accessed along with the user id and password to log into your data. To get your password press Get Passw button.
    • Resetting to defaults also changes your password.
  • Is Road checkbox if checked records road driving track.
  • Start on Reboot automatically starts the application on the device reboot.
  • Debug Log File checkbox if checked creates an application log file in FONT_Logfiles directory.
  • Cache if Clear button clicked removes all unsent trackpoints from smartphone cache. Send button uploads the cached trackpoints to the clouds if connection is available.
    • If the cache reads 0 no unsent trackpoints left on the phone.
Reporting issues/bugs:
  • Use Report Problem envelope icon on the bottom of Action bar.
